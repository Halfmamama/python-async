# Архитектурный срез фреймворка Multik (Crypto Airdrop Automation)

Этот каталог содержит обезличенный срез архитектуры моего фреймворка для параллельной мультиаккаунт-автоматизации взаимодействия с крипто-проектами (на примере проекта Nemesis) через веб-интерфейс кошелька MetaMask и анти-детект браузеры на базе асинхронного Playwright.

---

## Архитектура и ключевые инженерные решения

1. **Реестр проектов и паттерн Runner-адаптеров**  
   Фреймворк управляется единым оркестратором [launcher.py](./launcher.py), который считывает глобальную конфигурацию аккаунтов и предоставляет CLI-интерфейс. Каждый проект (например, [Nemesis](./Nemesis/)) подключается через легкий Runner-адаптер. Это позволяет изолировать логику конкретного тестнета от логики управления параллельными потоками, ретраями и обработкой сессий.

2. **WalletPolicy вместо глобального стейта транзакций**  
   Для управления действиями в кошельке MetaMask реализован класс [WalletPolicy](./core/wallet.py). Каждый скрипт перед началом работы декларирует политику (разрешенную сеть, лимиты газа и список разрешенных операций, таких как `connect`, `sign`, `approve` или `confirm`). Это гарантирует безопасность выполнения транзакций и предотвращает случайные действия на mainnet-сетях.

3. **Анти-детект и humanize-эмуляция**  
   Модуль [browser.py](./core/browser.py) настраивает отпечатки браузера Chromium (таймзоны, языки, геолокацию, параметры экрана) под каждый профиль. Модуль [humanize.py](./core/humanize.py) реализует случайные задержки между кликами и эмулирует плавное движение мыши (кривые Безье), снижая риск обнаружения автоматизации анти-фрод системами.

4. **Инструменты отказоустойчивости и отладки**  
   - Снятие скриншота экрана и сохранение полной копии HTML-кода страницы при падении сценария ([failure.py](./core/failure.py)).
   - Безопасные клики по элементам ([utils.py](./core/utils.py#L123-L131)) с перехватом исключений, предотвращающие падение скрипта при задержках отрисовки интерфейса.
   - Запись Playwright Tracing в ZIP-архив при обнаружении сбоев.

## Реальные истории из практики отладки (Debugging Stories)

### 1. Состояние гонки при разблокировке MetaMask под высокой нагрузкой
**Проблема**: При запуске более 4-х параллельных браузеров в режиме Playwright Tracing часть потоков зависала на этапе ввода пароля в MetaMask.
**Причина**: Включение трассировки (`tracing.start`) создавало высокую дисковую нагрузку при инициализации Chromium-контекста. MetaMask не успевал отрисовать форму ввода пароля до срабатывания дефолтного таймаута Playwright, что приводило к падению сессии по TimeoutError.
**Решение**: В [browser.py](./core/browser.py) был внедрен асинхронный мьютекс (`asyncio.Lock()`) для сериализации запуска браузеров. Браузерные контексты запускаются поочередно с небольшой staggered-задержкой (1-3 секунды), что разгружает диск и процессор, убирая состояние гонки при разблокировке кошелька.

### 2. Зависание процессов Chrome при закрытии контекста
**Проблема**: После завершения работы аккаунта метод `context.close()` периодически зависал навсегда, оставляя процессы Chrome "зомби" в системе, что приводило к утечке оперативной памяти под нагрузкой.
**Причина**: Playwright не всегда мог корректно закрыть Chromium, если в этот момент происходили внутренние сетевые запросы расширений кошельков (MetaMask / Rabby).
**Решение**: Я переопределил стандартный метод `context.close()` в [browser.py](./core/browser.py), обернув его в `asyncio.wait_for` с таймаутом в 8 секунд. При превышении таймаута вызывается вспомогательная утилита `force_kill_chrome_by_profile`, которая находит PID процесса Chrome по его флагу `--user-data-dir` и жестко завершает процесс через `psutil`.

## Ограничения демонстрационной версии

В данном репозитории представлен срез фреймворка исключительно в демонстрационных целях:
- Реальные файлы `config.py` с паролями, адресами кошельков и прокси исключены из Git и заменены на пример конфигурации [config.example.py](./config.example.py).
- Интеграция с базой данных sqlite и Google Sheets заменена на заглушки ([db.py](./core/db.py) и [sheets.py](./core/sheets.py)), возвращающие пустые или дефолтные значения.
